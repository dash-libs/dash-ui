# Security Policy

## Supported Versions

Only the latest published version on PyPI is supported. Patch releases ship
weekly (or on demand) via the automated release workflow.

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Instead, email the maintainer directly (see `authors` in `pyproject.toml`)
with:

- A description of the vulnerability and its potential impact
- Steps to reproduce, or a proof of concept
- Any suggested remediation, if you have one

We'll acknowledge your report and aim to ship a fix or mitigation within a
reasonable timeframe given this is a small, single-maintainer project.
